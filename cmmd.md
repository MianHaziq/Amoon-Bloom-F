Here’s a polished and professional AI prompt you can use to get much better results from an AI coding assistant or development agent:

You need to perform a complete end-to-end analysis of our entire project, including frontend, backend, APIs, authentication flow, admin panel, database integration, and state management architecture.

Your task is to thoroughly audit the application, identify all performance bottlenecks, architectural problems, bad coding practices, broken integrations, UI/UX inconsistencies, and logic issues, then fix them systematically one by one.

Main Issues to Focus On:

1. State Management

* Implement proper and scalable state management for users, authentication, sessions, roles, and application data.
* Remove unnecessary re-renders, duplicated states, prop drilling, and inconsistent data handling.
* Ensure frontend state stays synchronized with backend data properly.

2. Admin Panel Problems

* After entering admin credentials, the admin panel is not opening/loading correctly.
* Diagnose the complete authentication and routing flow.
* Fix login handling, token/session management, protected routes, role validation, and dashboard rendering.
* Ensure admin access works reliably and securely.

3. Performance & Responsiveness

* The entire application feels slow and poorly optimized.
* Identify all bad approaches currently used in the project.
* Optimize API calls, loading states, rendering cycles, database queries, component structure, caching, and async operations.
* Improve frontend responsiveness and backend efficiency.

4. Codebase Cleanup & Architecture

* Refactor poor-quality code and remove unnecessary complexity.
* Standardize folder structure, reusable components, services, hooks, utilities, and API layers.
* Follow professional development standards and best practices.

5. Backend Integration

* Verify that every frontend feature is properly integrated with the backend.
* Fix broken APIs, incorrect payloads, missing validations, error handling, and inconsistent responses.
* Ensure all CRUD operations work correctly.

6. Testing & Validation

* After every fix, test the functionality properly.
* Verify authentication flows, admin panel operations, user flows, API integrations, forms, and database interactions.
* Ensure there are no regressions or broken features after changes.

Execution Requirements:

* Analyze issues deeply before applying fixes.
* Fix issues step by step in a structured manner.
* Explain root causes and implemented solutions clearly.
* Prioritize stability, scalability, maintainability, and performance.
* Do not apply temporary hacks or shortcuts.
* Ensure the final system is production-ready and fully optimized.
